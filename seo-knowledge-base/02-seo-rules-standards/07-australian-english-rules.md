# Australian English Writing Rules — SecurityBlogs Australia

**Document type:** SEO Rules & Standards  
**Applies to:** All content published on SecurityBlogs — articles, service pages, meta data, schema, social copy, and email  
**Last reviewed:** June 2026  
**Owner:** Editorial Team

## Overview

SecurityBlogs is an Australian publication serving the Australian security industry. Every word published on the site must reflect Australian English — not American English, British English, or a mixed hybrid.

This matters for three reasons:

1. **Audience trust:** Australian readers notice Americanisms. A publication that writes "color" instead of "colour" or "organize" instead of "organise" signals to Australian readers that it is not genuinely local — undermining credibility and trust.

2. **SEO search intent alignment:** Australian users search in Australian English. A security guard company in Brisbane searching for "optimisation" will not find pages optimised for the American "optimization" spelling. While Google is increasingly good at matching variants, exact matches still carry weight — particularly in long-tail queries.

3. **Brand standards:** SecurityBlogs' brand voice is professional, locally grounded, and industry-specific. Australian English is non-negotiable.

## 1. Core Spelling Differences

The following table covers the most commonly encountered spelling differences between Australian/British English and American English. Australian English follows British spelling conventions in most cases.

### 1.1 -ise vs -ize (and -isation vs -ization)

Australian English uses **-ise** and **-isation** endings. American English uses **-ize** and **-ization**.

| Australian (Correct) | American (Incorrect) |
|---|---|
| Optimise | Optimize |
| Organise | Organize |
| Recognise | Recognize |
| Specialise | Specialize |
| Analyse | Analyze |
| Authorise | Authorize |
| Customise | Customize |
| Prioritise | Prioritize |
| Standardise | Standardize |
| Capitalise | Capitalize |
| Monetise | Monetize |
| Digitalise | Digitalize |
| Emphasise | Emphasize |
| Minimise | Minimize |
| Maximise | Maximize |
| Optimisation | Optimization |
| Organisation | Organization |
| Specialisation | Specialization |
| Recognition | Recognition (same — exception) |
| Authorisation | Authorization |

> **Note:** "Recognise" follows the -ise rule, but "recognition" is the same in both variants — it is an exception.

### 1.2 -our vs -or

Australian English retains the **-our** ending. American English drops the 'u'.

| Australian (Correct) | American (Incorrect) |
|---|---|
| Colour | Color |
| Favour | Favor |
| Behaviour | Behavior |
| Neighbour | Neighbor |
| Honour | Honor |
| Labour | Labor |
| Endeavour | Endeavor |
| Humour | Humor |
| Rigour | Rigor |
| Vigour | Vigor |

### 1.3 -re vs -er

Australian English uses **-re** endings. American English uses **-er**.

| Australian (Correct) | American (Incorrect) |
|---|---|
| Centre | Center |
| Metre | Meter |
| Theatre | Theater |
| Litre | Liter |
| Fibre | Fiber |
| Calibre | Caliber |

### 1.4 -ce vs -se (Noun/Verb Pairs)

In Australian English, the **noun** and **verb** forms are sometimes spelled differently. This is a common source of errors.

| Word | Noun (AU) | Verb (AU) | Notes |
|---|---|---|---|
| Licence / License | Licence | License | "A security licence" (noun); "to license a business" (verb) |
| Practice / Practise | Practice | Practise | "Best practice" (noun); "to practise" (verb) |
| Advice / Advise | Advice | Advise | "SEO advice" (noun); "we advise clients" (verb) |
| Device / Devise | Device (noun only) | Devise | "A security device"; "to devise a strategy" |

**Security industry examples:**
- "Holding a valid security **licence**" (noun — AU correct)
- "This **licences** the operator to provide services" (verb — AU correct)
- "Best **practice** in security management" (noun)
- "We **practise** ethical journalism" (verb)

### 1.5 -ll vs -l (Doubled Consonants)

Australian English generally doubles the 'l' before a suffix. American English does not.

| Australian (Correct) | American (Incorrect) |
|---|---|
| Travelling | Traveling |
| Modelling | Modeling |
| Fulfilling | Fulfilling (same) |
| Cancelled | Canceled |
| Labelling | Labeling |
| Signalling | Signaling |
| Counsellor | Counselor |
| Jeweller | Jeweler |

### 1.6 Other Common Differences

| Australian (Correct) | American (Incorrect) | Notes |
|---|---|---|
| Maths | Math | "Maths" is standard in AU |
| Programme | Program | Except for "computer program" — "program" is correct in tech contexts |
| Cheque | Check | Financial instrument |
| Tyre | Tire | Vehicle component |
| Catalogue | Catalog | |
| Dialogue | Dialog | Except in software UI context (dialog box is acceptable) |
| Ageing | Aging | |
| Judgement | Judgment | Both are used in AU; "judgement" is preferred in non-legal contexts |
| Defence | Defense | "Australian defence sector", "Defence Force" |

## 2. Date Formats

### 2.1 Standard Australian Date Format

Use **DD Month YYYY** format in running text:

- ✔ 15 June 2026
- ✔ 1 January 2026
- ✗ June 15, 2026 (American — comma placement and month-first)
- ✗ 15/06/2026 (numerical — acceptable in tables and metadata but not in body copy)
- ✗ 06/15/2026 (American numerical — never use)

### 2.2 Date in Metadata and Schema

For schema markup and technical metadata, use **ISO 8601 format**: `YYYY-MM-DD`

Example: `"datePublished": "2026-06-15"`

### 2.3 Date Ranges

- ✔ 10–15 June 2026 (en dash, no spaces)
- ✔ June to August 2026
- ✗ June 10-15, 2026 (American format)

### 2.4 Referring to Years and Decades

- ✔ "in the 2020s" (not "in the 2020's")
- ✔ "the 2025–26 financial year"
- ✔ "FY2025–26" (abbreviated financial year)

## 3. Currency

### 3.1 Australian Dollar Format

- Symbol: **$** (same as US but contextualised as AUD)
- When disambiguation is needed: **AUD $** or **A$**
- Format: **$1,500** (comma as thousands separator, full stop as decimal)
- ✔ "$2,500 per month"
- ✔ "AUD $5,000"
- ✗ "$2.500" (European format — incorrect in Australia)
- ✗ "USD $2,500" for Australian prices

### 3.2 Price Ranges

- ✔ "$500–$2,000 per month"
- ✔ "from $500/month"
- ✗ "$500-$2,000" (hyphen instead of en dash)

### 3.3 GST

When referencing prices in Australian content, clarify GST status where relevant:

- "All prices are inclusive of GST"
- "Prices exclude GST — GST will be added at checkout"
- GST = Goods and Services Tax (10% in Australia)

## 4. Phone Number Format

### 4.1 International Format (for contact pages, footer, schema)

Use the international dialling format for all contact numbers on the website:

**Format:** `+61 X XXXX XXXX`

- ✔ `+61 3 9000 0000` (Melbourne landline)
- ✔ `+61 4XX XXX XXX` (mobile)
- ✔ `+61 2 9000 0000` (Sydney landline)

### 4.2 Local Format (within Australian content text)

When writing phone numbers within Australian body content:

- ✔ `03 9000 0000` (Melbourne)
- ✔ `0412 345 678` (mobile)
- ✗ `(03) 9000-0000` (American format with hyphens)
- ✗ `39000000` (no spaces — hard to read)

### 4.3 Toll-Free Numbers

- ✔ `1300 XXX XXX`
- ✔ `1800 XXX XXX`
- Format with spaces, no hyphens

## 5. Address Format

### 5.1 Australian Standard Address Format

```
[Unit/Suite Number] [Street Number] [Street Name] [Street Type]
[Suburb] [State Abbreviation] [Postcode]
Australia
```

**Example:**
```
Suite 4, 123 Collins Street
Melbourne VIC 3000
Australia
```

### 5.2 State Abbreviations (Standard)

| State / Territory | Abbreviation |
|---|---|
| Victoria | VIC |
| New South Wales | NSW |
| Queensland | QLD |
| South Australia | SA |
| Western Australia | WA |
| Tasmania | TAS |
| Australian Capital Territory | ACT |
| Northern Territory | NT |

### 5.3 Referring to States in Content

- ✔ "businesses in Victoria" (lowercase 'businesses')
- ✔ "across New South Wales and Queensland"
- ✔ "Queensland security companies" (state as adjective)
- ✗ "businesses in VIC" (use state abbreviations only in addresses and headings, not in body prose)

## 7. Security Industry Terminology — Australian Usage

The Australian security industry has specific terminology that differs from the UK and US. Content writers must use the correct Australian industry terms.

### 7.1 Industry Body and Regulatory References

| Term | Notes |
|---|---|
| **ASIAL** | Australian Security Industry Association Limited — Australia's peak security industry body |
| **ACSC** | Australian Cyber Security Centre |
| **AGSVA** | Australian Government Security Vetting Agency |
| **ASIO** | Australian Security Intelligence Organisation |
| **AFP** | Australian Federal Police |
| **State police** | Each state has its own police force (e.g., Victoria Police, NSW Police Force) |
| **Security licence** | Always "licence" (noun) — e.g., "a Class 1 security licence" |
| **Master Licence** | Held by security companies (not "license") |

### 7.2 Licence Classes (Australian Context)

These vary by state — always specify the state when referencing specific licence classes. Generic references:

- "Crowd controller licence"
- "Bodyguard licence"
- "Guard dog handler licence"
- "Security consultant licence"

### 7.3 Industry Terms — Australian vs Other Markets

| Australian Term | UK/US Equivalent | Notes |
|---|---|---|
| Security officer | Security guard | "Security officer" is the preferred professional term in AU |
| Crowd controller | Door supervisor / Bouncer | AU industry and legislative term |
| Monitoring centre | Monitoring station / Alarm receiving centre | |
| CCTV | CCTV / Surveillance camera | Same acronym; fine in both formats |
| Duress alarm | Panic button / Personal alarm | |
| Cash-in-transit (CIT) | Armoured transport / Cash transport | |
| Patrol officer | Patrol guard | |
| Control room operator | Dispatcher / Control room operative | |
| Principal licensee | Master licensee / License holder | |

### 7.4 Legislation References (Australian)

When referencing legislation in content, use the full Australian legal citation:

- ✔ "*Security Industry Act 1997* (NSW)"
- ✔ "*Private Security Act 2004* (VIC)"
- ✔ "*Security Providers Act 1993* (QLD)"
- Always italicise Act titles
- Always specify the jurisdiction in parentheses

## 8. Punctuation and Style Conventions

### 8.1 Quotation Marks

Australian English uses **single quotation marks** as the primary style, with double quotes for quotes within quotes. However, for digital content, double quotes are acceptable and often preferred for readability.

- ✔ 'security industry' (AU traditional)
- ✔ "security industry" (acceptable in digital content)
- Either is acceptable — choose one and be consistent within each piece

### 8.2 Oxford Comma

The Oxford comma (serial comma) is optional in Australian English but recommended for clarity in lists of three or more items.

- ✔ "SEO, AEO, and GEO services" (with Oxford comma — recommended)
- ✔ "SEO, AEO and GEO services" (without — also acceptable)

### 8.3 Em Dash vs En Dash

- **Em dash (—):** Used for parenthetical asides — like this one — or to introduce an explanation. No spaces in AU style: `word—word`
- **En dash (–):** Used for ranges: "2020–2026", "$500–$2,000"
- **Hyphen (-):** Used to join compound modifiers: "well-established company", "industry-specific SEO"

### 8.4 Numerals

- Spell out numbers one through nine in body copy
- Use numerals for 10 and above
- Exception: always use numerals before units of measurement ("3 months", "5 km", "2 years")
- Use commas for thousands: 10,000 not 10000
- Percentages: use the % symbol in body copy ("15%", not "fifteen per cent" — exception for formal/legal documents)

## 9. Quick Reference Checklist

Before publishing any content, verify:

- [ ] All -ise/-isation spellings used (not -ize/-ization)
- [ ] All -our spellings used (not -or)
- [ ] All -re spellings used (not -er) where applicable
- [ ] Licence (noun) and practise (verb) spelled correctly in context
- [ ] Dates written as DD Month YYYY in body copy
- [ ] Schema dates in ISO 8601 format
- [ ] Currency shown as $ or AUD $ with comma as thousands separator
- [ ] Phone numbers in +61 format on contact/schema elements
- [ ] Address uses Australian state abbreviations and postcode
- [ ] "Mobile" used instead of "cell"
- [ ] "Get in touch" used instead of "reach out" where relevant
- [ ] Security industry terms are Australian (e.g., "security officer", "crowd controller", "security licence")
- [ ] Any legislation referenced includes jurisdiction in parentheses
