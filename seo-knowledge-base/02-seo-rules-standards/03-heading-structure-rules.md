**Document type:** SEO Rules & Standards  
**Applies to:** All pages published on SecurityBlogs  
**Last reviewed:** June 2026  
**Owner:** SEO Team

---

## Overview

Heading tags (H1 through H6) serve two critical functions: they communicate the structure and hierarchy of a page's content to both users and search engines, and they present opportunities to capture featured snippets, People Also Ask (PAA) results, and AI-generated answer citations.

Correct heading structure is foundational to SEO, AEO (Answer Engine Optimisation), and GEO (Generative Engine Optimisation). Pages with poor or absent heading structure are harder for Google to understand and far less likely to win featured placements.

---

## 1. H1 — Page Title

### Rule: One H1 per page. No exceptions.

The H1 is the primary heading of the page. It signals to search engines what the page is fundamentally about. Multiple H1 tags create ambiguity and dilute the primary topic signal.

### H1 Requirements:

- **One H1 per page** — never use more than one
- **Primary keyword must appear in the H1** — ideally near the beginning
- The H1 should be **closely related to but not necessarily identical to the meta title** — the meta title is written for the SERP; the H1 is written for the reader on the page
- The H1 is always the first visible heading on the page
- H1s must not be used decoratively (e.g., inside call-to-action boxes, sidebars, or footers)
- Aim for **40–70 characters** — long enough to be descriptive, short enough to avoid dilution
- Do not stuff multiple keywords into the H1

### H1 by Page Type:

| Page Type | H1 Example |
|---|---|
| Homepage | `Expert Digital Marketing for Australian Security Businesses` |
| SEO Service Page | `SEO Services for Security Companies in Australia` |
| Blog Post | `How to Rank Your Security Company on Google in Australia` |
| Category Page | `SEO Articles and Guides for the Security Industry` |
| Location Page | `Security Industry SEO Services in Melbourne` |
| FAQ Page | `Frequently Asked Questions About Security Business SEO` |

### H1 vs Meta Title — Difference Explained:

The meta title is constrained to 50–60 characters and must end with `| SecurityBlogs`. The H1 is visible on the page and can be slightly longer, more conversational, or more descriptive. They should be closely aligned but need not be identical.

**Meta Title:** `SEO for Security Companies in Australia | SecurityBlogs`  
**H1:** `SEO Services for Australian Security Companies — Grow Your Online Presence`

## 2. H2 — Main Section Headings

H2 tags define the primary sections of a page. Each H2 represents a distinct subtopic that supports the page's main theme (defined by the H1).

### H2 Requirements:

- Use **one H2 per major section** of content
- Include **secondary or supporting keywords** naturally in H2s where appropriate
- H2s are the primary target for **featured snippets** — write them as clear, question-answering headings where relevant
- Aim for **40–80 characters**
- H2s should follow a logical progression through the page topic
- Do not skip from H1 to H3 — H2 must always precede H3

### H2 Keyword Strategy:

- Target **LSI (Latent Semantic Indexing) keywords** — related terms Google associates with the primary topic
- Use **question-format H2s** for content targeting PAA boxes (e.g., "What Is…", "How Does…", "Why Do Security Companies Need…")
- Use **statement-format H2s** for definitive sections (e.g., "Benefits of SEO for Security Firms", "How We Approach Security Industry SEO")

### H2 for Featured Snippets and AEO:

To win featured snippets and AI answer citations, H2s (and H3s) should be structured as direct questions or clear topic statements, with the answer appearing in the first 40–60 words immediately beneath the heading.

**Example structure:**
```
## What Is AEO and How Does It Differ from SEO?

Answer Engine Optimisation (AEO) is the practice of optimising content 
so it is selected as the direct answer by AI-powered search tools and voice 
assistants. Unlike traditional SEO, which focuses on ranking in the blue links, 
AEO targets the zero-click answer position — the response displayed before 
any ranked results.
```

## 3. H3 — Subsection Headings

H3 tags define subsections within an H2 section. They are used to break down a major section into specific points, steps, or sub-topics.

### H3 Requirements:

- H3 must always appear **within an H2 section** — never jump from H2 to H3 without content between them
- Use H3 for **step-by-step content**, **lists of specific points**, **examples**, or **sub-categories**
- Include long-tail keywords in H3s where natural — these are particularly effective for question-based long-tail queries
- Aim for **30–70 characters**
- H3s within FAQ sections are extremely powerful for PAA (People Also Ask) placement

### H3 for PAA and Long-Tail Queries:

Write H3s as the exact question a user might ask. Google uses these heading-answer pairs to populate PAA boxes.

**Example:**
```
### How Long Does It Take to See SEO Results for a Security Company?

Most security companies see meaningful improvements in organic rankings 
within 3 to 6 months of implementing a structured SEO strategy. Highly 
competitive markets such as Sydney or Melbourne may require 6 to 12 months 
for page-one placement on primary keywords.
```

## 4. H4 — Sub-subsection Headings

H4 tags are used to further subdivide H3 sections. They are appropriate for detailed, long-form content (2,000+ words) where a subsection genuinely contains multiple distinct components.

### H4 Requirements:

- Use sparingly — not every page needs H4 tags
- H4 must always appear within an H3 section
- Appropriate for: step-by-step tutorials, technical guides, multi-part comparisons, comprehensive resource pages
- Aim for **20–60 characters**

**Example use case:** An in-depth guide to SEO for security companies might have:
```
## On-Page SEO for Security Websites
### Optimising Service Pages
#### Writing SEO-Friendly Service Descriptions
#### Choosing the Right Keywords for Security Services
```

---

## 5. H5 and H6

H5 and H6 are rarely needed in standard web content. Their use at SecurityBlogs is limited to:

- Highly technical reference documents
- Legal or compliance documents with deep nesting structures
- Large resource pages with six or more levels of hierarchy

**Rule:** Do not use H5 or H6 unless content structure genuinely requires six levels of hierarchy. Artificially deep nesting makes content harder to read and provides no meaningful SEO benefit.

## 6. Heading Length Guidelines

| Heading Level | Recommended Length | Maximum |
|---|---|---|
| H1 | 40–70 characters | 80 characters |
| H2 | 40–80 characters | 90 characters |
| H3 | 30–70 characters | 80 characters |
| H4 | 20–60 characters | 70 characters |
| H5 | 15–50 characters | 60 characters |

**Important:** These are guidelines, not hard rules enforced by Google. A heading that is genuinely clear and descriptive at 85 characters is better than a forced, vague heading at 55 characters. Clarity and keyword relevance take priority over character count.

## 7. Heading Structure for AEO and Featured Snippets

To maximise featured snippet and AI citation opportunities, the following heading-content pattern should be used for informational pages and blog posts:

### The Question-Answer Pattern:

```
## [Question as H2]
[Direct, concise answer in 40–60 words immediately below]
[Supporting detail below the direct answer]
```

### The Definition Pattern (for "What is" queries):

```
## What Is [Topic]?
[Topic] is [definition in 1–2 sentences]. [Additional context].
```

### The How-To Pattern (for process queries):

```
## How to [Task]

### Step 1: [First Action]
[Explanation]

### Step 2: [Second Action]
[Explanation]
```

### The List Pattern (for "best of" or "types of" queries):

```
## [Number] Types of [Topic] for Security Companies

### 1. [First Item]
[Brief explanation]

### 2. [Second Item]
[Brief explanation]
```

## 9. Common Heading Mistakes to Avoid

| Mistake | Why It's a Problem | Correction |
|---|---|---|
| Multiple H1 tags on one page | Dilutes the primary topic signal; confuses Google | Remove all H1s except the first |
| Skipping heading levels (H1 → H3) | Breaks semantic structure; poor accessibility | Always use sequential heading levels |
| Using headings for styling only | Misrepresents page structure to search engines | Use CSS for styling; use headings only for structure |
| Keyword-stuffing headings | Looks spammy; can trigger quality penalties | One focus keyword per heading, naturally placed |
| All headings as questions | Unbalanced structure; harder to read | Mix question and statement headings appropriately |
| Headings longer than 90 characters | Dilutes SEO value; harder to scan | Trim to the most important phrase |
| No headings at all | Page is unstructured; cannot win featured snippets | Add appropriate H1, H2, and H3 structure |

## 10. Heading Quality Checklist

Before publishing any page, verify:

- [ ] Exactly one H1 is present on the page
- [ ] H1 contains the primary keyword near the beginning
- [ ] H1 is between 40 and 70 characters
- [ ] H2s are used for all major sections
- [ ] H3s appear only within H2 sections
- [ ] No heading levels are skipped
- [ ] At least two H2s use question format (for AEO/featured snippet targeting)
- [ ] Each heading-content block has an answer in the first 60 words beneath it (for featured snippet pages)
- [ ] Australian English spelling is used in all headings
- [ ] No keyword stuffing across headings
- [ ] Heading hierarchy forms a logical outline of the page content
