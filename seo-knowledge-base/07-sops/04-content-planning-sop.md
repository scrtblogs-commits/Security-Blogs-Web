# SOP 04 — Content Planning

**Organisation:** SecurityBlogs Australia
**Department:** SEO / Digital Marketing
**Document Version:** 1.0
**Last Reviewed:** June 2026
**Review Frequency:** Monthly (editorial calendar); Quarterly (strategy review)

---

## Purpose

To provide a structured process for building and maintaining the SecurityBlogs Australia monthly editorial calendar, ensuring every piece of content is aligned to a keyword strategy, topic cluster, business objective, and publish schedule. This SOP ensures content is planned proactively, strategically, and with clear ownership before any writing begins.

---

## Scope

Applies to all content produced for SecurityBlogs Australia including blog posts, pillar pages, guides, industry news articles, sponsored content, and resource pages. Relevant to the SEO strategist, content team lead, and editorial team. The monthly editorial calendar is produced in the last week of the preceding month.

## Step-by-Step Process

### Phase 1 — Input Gathering

1. Before building the editorial calendar, collect the following inputs:
   - **Keyword Map** (SOP 02): Priority 1 and Priority 2 keywords requiring new content
   - **Competitor Content Gap** (SOP 03): Identified content gaps
   - **GSC Performance Report** (SOP 10): Underperforming pages that may need refreshes rather than new content
   - **Client/Stakeholder Requests:** Any sponsored, promotional, or priority topics requested by clients or management
   - **Industry Calendar:** Upcoming security industry events, award seasons, regulatory changes (e.g., new licensing laws, ASIAL events, trade shows)
2. Consolidate all inputs into a **Content Ideas Pool** tab in the editorial calendar spreadsheet.

### Phase 2 — Topic Cluster Mapping

3. Review the SecurityBlogs topic cluster architecture. The site should be structured around core pillar topics with supporting cluster content. Example cluster structure:
   - **Pillar:** Physical Security
     - Cluster: Access Control Systems, CCTV, Security Guards, Alarm Systems, Perimeter Security
   - **Pillar:** Cybersecurity for Security Businesses
     - Cluster: Network Security, Data Protection, Compliance, Incident Response
   - **Pillar:** Security Industry Australia
     - Cluster: ASIAL news, licensing, training, industry events, regulations
   - **Pillar:** Security Technology
     - Cluster: AI in security, smart home security, body-worn cameras, drones
4. For each content idea in the pool, assign it to a **Topic Cluster** and **Sub-Cluster**.
5. Identify if any topic cluster lacks a pillar page. If so, pillar page creation takes precedence over cluster content.

### Phase 3 — Content Type Decision

6. For each content idea, determine the appropriate **Content Type** based on search intent and strategic purpose:

| Content Type | When to Use | Typical Word Count |
|---|---|---|
| **Pillar Page** | Covers a broad topic comprehensively; links to all cluster content | 3,000–6,000 words |
| **Cluster Blog Post** | Covers a specific sub-topic; links back to pillar | 1,200–2,500 words |
| **Long-Form Guide** | In-depth educational content; high-value informational intent | 2,500–5,000 words |
| **Industry News Post** | Timely news reporting; lower word count acceptable | 400–800 words |
| **Opinion / Analysis** | Expert commentary; thought leadership | 800–1,500 words |
| **Sponsored / Partner Content** | Commercially supported content; must be labelled | 600–1,500 words |
| **Resource / Tool Page** | Static resource (checklist, template, calculator) | Varies |
| **FAQ Page** | Answers cluster of related questions; targets PAA | 800–2,000 words |

7. Record the content type in the editorial calendar.

### Phase 4 — Keyword Assignment

8. For each content piece, assign:
   - **Primary Keyword:** One keyword this page is primarily targeting. Must be from the approved Keyword Map (SOP 02).
   - **Secondary Keywords:** 3–6 related keywords and LSI terms to be incorporated naturally into the content.
   - **Target Search Intent:** Informational / Commercial / Transactional / Navigational.
9. Confirm that the primary keyword is not already assigned to another page (check the Keyword Map for cannibalisation).
10. Record keyword assignments in the editorial calendar.

### Phase 5 — Word Count Target Setting

11. Determine the target word count by:
    - Reviewing the average word count of the top 5 ranking pages for the primary keyword (use Ahrefs or manually check)
    - Setting the target at the average or 10–20% above the average to provide comprehensive coverage
    - Applying the content type minimum guidelines from Phase 3
12. Record the word count target in the editorial calendar.

### Phase 6 — Internal Linking Plan

13. For each planned content piece, identify:
    - **Links FROM this page:** Which existing pillar or cluster pages should this new content link to? (minimum 3 internal links)
    - **Links TO this page:** Which existing pages should be updated to link to this new page once published? (minimum 2 existing pages)
14. Record the internal link plan in the brief template (see below). This pre-planning prevents orphan pages.

### Phase 7 — Target SERP Feature

15. Check the SERP for the primary keyword and identify which features are present:
    - If a **Featured Snippet** is present: plan the content to include a direct, concise answer (40–60 words) early in the post and structure content with clear question-based H2/H3 headings.
    - If **People Also Ask** is prominent: plan an FAQ section addressing the PAA questions.
    - If a **Video Carousel** is present: consider whether a companion video asset is feasible.
    - If an **Image Pack** is shown: plan to include original, descriptive images with keyword-rich alt text.
16. Record the target SERP feature in the editorial calendar and flag in the brief.

### Phase 8 — CTA Assignment

17. Assign a **primary Call to Action (CTA)** to each content piece based on funnel stage:

| Content Type / Intent | CTA Options |
|---|---|
| Informational blog post | Newsletter subscription, Related post, Download resource |
| Commercial guide | Contact form, Request quote, Service page link |
| Industry news | Newsletter subscription, Share on LinkedIn |
| Pillar page | Contact form, Download guide, Explore services |
| Sponsored content | Client-specified CTA (link to client product/service) |

18. Record the CTA in the brief.

### Phase 9 — Publish Schedule

19. Assign a **target publish date** to each content piece based on:
    - Production capacity (writing, editing, design, SEO review)
    - Industry events or seasonality (prioritise time-sensitive content)
    - Client commitments and deadlines
    - A target cadence of at least 8–12 blog posts per month for SecurityBlogs Australia
20. Assign content to writers. Each brief must be sent to the writer at least **10 business days** before the publish date.
21. Record publish dates and assigned writers in the editorial calendar.

### Phase 10 — Brief Creation

22. For every planned content piece, create a **Content Brief** using the template below.
23. The brief must be completed by the SEO strategist or content team lead before being passed to the writer.
24. Brief includes: keyword data, search intent, outline structure, target SERP feature, word count, internal links, CTA, reference URLs, tone guidance, and Australian English requirements.
25. Save the brief in Google Docs and link it from the editorial calendar.

## Tone and Style
- Australian English throughout (use "organisation" not "organization", "licence" not "license", "-ise" not "-ize" suffixes)

- Avoid excessive jargon; define technical terms on first use
- Active voice preferred
- No fluff or padding — every paragraph must add value

## Quality Checklist

- [ ] All content ideas sourced from Keyword Map, Competitor Gap, or GSC data
- [ ] Every content piece assigned to a topic cluster
- [ ] Content type determined and documented for each piece
- [ ] Primary and secondary keywords assigned (no cannibalisation)
- [ ] Word count targets set based on SERP analysis
- [ ] Internal link plan completed for each piece
- [ ] Target SERP feature identified
- [ ] CTA assigned to each piece
- [ ] Publish dates set with realistic lead times (10+ business days for writers)
- [ ] Content brief created and linked from calendar for every piece
- [ ] Editorial calendar shared with all relevant team members
- [ ] Sponsored or client-driven content flagged separately

## Common Mistakes to Avoid

- **Creating briefs without keyword data** — every piece of content must have a primary keyword before briefing begins. Content without keyword intent has no measurable SEO goal.
- **Assigning the same primary keyword to multiple articles** — review the keyword map before assigning.
- **Building orphan pages** — always plan internal links before publishing. A page with no internal links pointing to it will struggle to gain PageRank regardless of content quality.
- **Overloading the editorial calendar** — an overpacked calendar leads to rushed content and quality drops. Capacity planning is as important as strategy.
- **Ignoring seasonal trends** — use Google Trends (AU) to ensure time-sensitive security topics (e.g., end-of-financial-year business decisions, summer construction season) are planned ahead of peak demand.
- **Skipping the SERP feature check** — without knowing what Google is displaying for a keyword, it is impossible to structure content to compete for features.
