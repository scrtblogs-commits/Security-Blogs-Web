# SOP: Keyword Research — SecurityBlogs Standard

**Version:** 1.0  
**Applies to:** SecurityBlogs content strategy and all security client SEO campaigns  
**Frequency:** At campaign start; quarterly review; before each content planning cycle

---

## PURPOSE

Keyword research identifies the search queries that target audiences use at each stage of the buying journey. This SOP ensures a repeatable, thorough process that produces a prioritised, actionable keyword list for any security business.

---

## TOOLS REQUIRED

- Google Keyword Planner (free, requires Google Ads account)
- Ahrefs Keywords Explorer (preferred) or SEMrush
- Google Search Console (for existing site data)
- Ahrefs Site Explorer (for competitor keyword analysis)
- AnswerThePublic or AlsoAsked.com (for question-based keywords)
- Google Search (for PAA and related searches)
- SecurityBlogs Keyword Database (04-Keyword-Database folder — use as starting point)

---

## PHASE 1: DEFINE KEYWORD SCOPE

### Step 1: Document the business context
Before starting keyword research, record:
- Business type (installer, integrator, SaaS vendor, manufacturer, consultant)
- Services offered (specific products: CCTV, access control, alarms, etc.)
- Geographic target (city, state, national, international)
- Client size / target market (SME, commercial, enterprise, government)
- Competitive position (new entrant, established local leader, national brand)

### Step 2: Build a seed keyword list
Start with the SecurityBlogs Keyword Database (04-Keyword-Database) and extract all relevant terms for this business type. Then expand by brainstorming:
- Product and service names the business offers
- Problem words (what problems do their customers search about?)
- Location modifiers (suburb, city, state)
- Intent modifiers (installer, company, service, cost, quote, near me)

**Typical seed list: 20–50 keywords before expansion**

---

## PHASE 2: KEYWORD EXPANSION

### Step 3: Expand in Ahrefs Keywords Explorer
For each seed keyword:
1. Enter into Ahrefs Keywords Explorer
2. Navigate to "Matching terms" and "Related terms" tabs
3. Export all results with: keyword, monthly search volume, keyword difficulty (KD), clicks, CPC
4. Filter out irrelevant or off-topic terms

### Step 4: Check competitor keywords
1. Enter the top 3 competitor domains into Ahrefs Site Explorer
2. Navigate to "Organic keywords" for each competitor
3. Export keywords driving traffic to competitor sites
4. Cross-reference against your seed list and add any missing relevant terms

### Step 5: Extract question keywords
1. Use AnswerThePublic or AlsoAsked.com for each seed keyword
2. Record all question-format queries (What, How, Why, When, Where, Which)
3. These become FAQ, blog post, and People Also Ask targets

### Step 6: Extract Google-suggested keywords
For each seed keyword:
1. Search Google and capture the "People Also Ask" box questions
2. Scroll to the bottom and capture "Related searches"
3. These are Google's own signals of related intent

---

## PHASE 3: KEYWORD ORGANISATION

### Step 7: Build the keyword matrix
Create a spreadsheet with these columns:
- Keyword
- Monthly search volume (AU / US / GB as relevant)
- Keyword difficulty (0–100)
- CPC (cost per click — proxy for commercial intent)
- Intent (Informational / Commercial / Transactional)
- Content type (service page / blog post / FAQ / location page)
- Existing page (if a page already targets this keyword — for cannibalisation check)
- Priority (P1/P2/P3/P4)
- Notes

### Step 8: Group keywords by topic cluster
Organise keywords into the content clusters defined in 06-Content-Clusters:
- Security SEO cluster
- AEO cluster
- GEO cluster
- AI Visibility cluster
- Security Marketing cluster
- Guest Posting cluster
- Cybersecurity PR cluster

Add any new clusters identified during research.

### Step 9: Map keywords to pages (one primary keyword per page)
Every page targets one primary keyword and 3–5 secondary keywords:
- **Primary keyword** — the main ranking target for the page
- **Secondary keywords** — related terms used naturally in the content
- **No two pages should share the same primary keyword** — this causes cannibalisation

---

## PHASE 4: PRIORITISATION

### Step 10: Apply the CART framework
Prioritise keywords using the CART framework:

| Factor | Question | Weight |
|---|---|---|
| **Commercial value** | How closely does this keyword relate to a revenue-generating service? | 30% |
| **Achievability** | Given current domain authority, can we realistically rank for this? (KD vs. our DR) | 30% |
| **Relevance** | Does this keyword precisely match the target audience and their intent? | 25% |
| **Traffic potential** | What is the realistic click volume if we rank in position 1–3? | 15% |

Score each keyword 1–5 on each factor. Multiply by weight. Top-scoring keywords = P1 priorities.

### Step 11: Set the priority tier
- **P1** — High commercial value + achievable + highly relevant + decent traffic. Target immediately.
- **P2** — High relevance + decent traffic + achievable. Target within 30 days.
- **P3** — Informational + builds topical authority. Add to content calendar.
- **P4** — High competition / aspirational targets. Build toward over 6–12 months.

---

## PHASE 5: DELIVERABLE

### Keyword Research Report
Deliver to the client or use internally as a content brief, containing:

1. **Summary table** — top 20 priority keywords with volume, KD, intent, and content type
2. **Full keyword matrix** (spreadsheet — all keywords with all data)
3. **Content gap analysis** — priority keywords with no existing page targeting them
4. **Cannibalisation report** — keywords where multiple pages are competing
5. **Quick wins** — low-KD, decent-volume keywords that can rank quickly
6. **Long-term targets** — high-value keywords that require sustained effort

---

## PHASE 6: ONGOING MANAGEMENT

### Quarterly Keyword Review
Every 3 months, run a keyword review:
1. Check current rankings for all P1 and P2 keywords via Ahrefs or GSC
2. Identify: Which keywords have moved up (capitalise)? Which have dropped (investigate)?
3. Add: New keywords identified through GSC ("queries" report — search terms generating impressions but no clicks)
4. Update: The priority matrix based on new data

### Google Search Console — Ongoing Keyword Mining
Monthly:
1. Export all queries from GSC (Performance → Search Results → Queries)
2. Filter by: Impressions > 50, Position > 10 (ranking but not on page 1)
3. These are "low-hanging fruit" — content updates or on-page fixes can often move these to page 1
4. Add to the keyword matrix as P2 targets

## COMMON KEYWORD RESEARCH MISTAKES TO AVOID

| Mistake | Correct Approach |
|---|---|
| Targeting only high-volume keywords | Balance high-volume with long-tail; long-tail converts better |
| Ignoring KD | A KD of 60+ is very hard to rank for without significant DA |
| No intent matching | If keyword intent is informational, don't target it with a service page |
| Targeting the same keyword on multiple pages | One primary keyword per page — always |
| Ignoring location modifiers | For local businesses, geo-modified keywords are the most valuable |
| Not checking competitor keywords | Competitor gap analysis often reveals the best quick-win opportunities |
| Updating keywords without updating pages | Keyword research is useless without matching content creation or optimisation |
