# SOP: Content Planning & Competitor Analysis — SecurityBlogs Standard

**Version:** 1.0  
**Applies to:** SecurityBlogs content strategy and all active security client campaigns

---

# PART A: CONTENT PLANNING SOP

## PURPOSE

Content planning ensures every piece of content published has a clear keyword target, serves a specific audience intent, fits within a topic cluster, and advances the overall SEO and authority strategy.

---

## QUARTERLY CONTENT PLANNING CYCLE

Content planning runs on a 90-day cycle:

| Month | Activity |
|---|---|
| Month 1 | Plan and brief all content for the quarter |
| Month 2 | Publish planned content; begin next-quarter research |
| Month 3 | Evaluate performance; adjust quarter-end content; begin next-quarter planning |

---

## CONTENT PLANNING PROCESS

### Step 1: Review existing content performance (30 minutes)
Before planning new content, review what's already working:
1. Open Google Search Console → Performance → Pages
2. Record: Top 10 pages by clicks
3. Record: Top 10 pages by impressions (especially those not in the top 10 by clicks — these are quick-win candidates)
4. Record: Pages that have declined in clicks vs. the previous quarter
5. Identify: Are there existing pages that should be updated/expanded rather than creating new content?

**Content update priority:** Updating an existing page that ranks on page 2–3 is often faster ROI than creating a new post.

---

### Step 2: Review the content cluster map (30 minutes)
Open the Content Cluster Master (06-Content-Clusters/content-cluster-master.md):
1. Which pillar pages have NOT yet been published?
2. Which clusters have fewer than 3 supporting posts live?
3. Which supporting post ideas from the cluster map are not yet covered?

**Priority rule:** Publish pillar pages before supporting pages. A cluster without a pillar is architecturally incomplete.

---

### Step 3: Keyword gap analysis (30 minutes)
1. Run the Keyword Research SOP (07-SOPs/keyword-research-sop.md) for any new keyword areas
2. Cross-reference the keyword database against published content
3. Identify: P1 and P2 keywords with no existing page targeting them
4. These become new content briefs

---

### Step 4: Competitive content gap (20 minutes)
For 3 key competitors:
1. Export their top content from Ahrefs (Site Explorer → Top Pages)
2. Identify: What are their top-traffic pages?
3. Identify: Are there topics they rank for that SecurityBlogs/the client is not covering?
4. Add: Missing high-value topics to the content calendar

---

### Step 5: Build the content calendar
Using a spreadsheet or project management tool, record for each planned piece:
- Title (working)
- Target keyword (primary)
- Secondary keywords
- Content type (blog / pillar / service page / FAQ / location page)
- Cluster assignment
- Target URL (planned)
- Word count target
- Template to use (from 03-Blog-Templates)
- Internal links planned (minimum 3)
- Schema types required
- Target publish date
- Assigned writer
- Status (planned / briefed / writing / editing / scheduled / published)

**Target content volume per quarter:**
- SecurityBlogs site: 12–18 new pieces (pillar + supporting mix)
- Active client campaigns: 4–8 new pieces per month depending on package

---

### Step 6: Brief each content piece
For each planned piece, create a written brief containing:
- Target keyword (primary)
- Secondary keywords
- SERP analysis (what ranks, what format, what headings, what word count)
- Audience and intent
- Key points to cover (pulled from SERP analysis + keyword research)
- Template to use
- Internal links to include
- CTA destination
- Word count
- Schema types
- Assigned to + deadline

---

## CONTENT CALENDAR MANAGEMENT

### Weekly content check-in
Every Monday, review:
- What content is due this week?
- Is it on track?
- Any blockers? (Subject matter expert unavailable, data needed, screenshot required)

### Monthly content review
First week of each month:
- How many pieces were published vs. planned? (Track consistency)
- How are last month's published pieces performing in GSC after 4 weeks?
- Are there any planned pieces that should be prioritised differently?

# PART B: COMPETITOR ANALYSIS SOP

## PURPOSE

Regular competitor analysis identifies ranking opportunities, content gaps, link building opportunities, and strategic threats in the security marketing space.

---

## QUARTERLY COMPETITOR ANALYSIS PROCESS

### Step 1: Identify the current top competitors (15 minutes)
For each primary keyword cluster, identify who ranks on page 1:
1. Search the target keyword in Google (use incognito mode)
2. Record the top 10 results
3. Identify: Which domains consistently appear across multiple keyword searches?
4. These are the active SEO competitors for this keyword cluster

**Do this for 5–10 priority keywords to build a list of 3–5 key competitors.**

---

### Step 2: Domain authority comparison (15 minutes)
For each identified competitor, record in Ahrefs:
- Domain Rating (DR)
- Number of referring domains
- Monthly organic traffic (estimated)
- Number of organic keywords ranking

Compare to SecurityBlogs / client site and record the gap.

---

### Step 3: Top pages analysis (20 minutes)
For each competitor:
1. Ahrefs Site Explorer → Top Pages (by organic traffic)
2. Export top 20 pages
3. Identify: What content is driving the most traffic?
4. Identify: Are any of these topics not yet covered by SecurityBlogs/client?
5. Add: Content gaps to the content calendar

---

### Step 4: Keyword gap analysis (20 minutes)
1. Ahrefs → Competitive Analysis → Content Gap
2. Enter SecurityBlogs / client domain + 3 competitor domains
3. Filter: Keywords competitors rank for that the target site does not
4. Sort by: Search volume
5. Export top 30–50 keywords
6. Add: High-value gaps to keyword database and content calendar

---

### Step 5: Backlink gap analysis (20 minutes)
1. Ahrefs → Competitive Analysis → Link Intersect
2. Enter competitor domains
3. Identify: Domains that link to 2+ competitors but NOT to the target site
4. These are priority link building targets — they have a pattern of linking to security/SEO content
5. Add top 10–20 link prospects to the outreach tracker

---

### Step 6: Content quality assessment (30 minutes)
For the top 5 competitor pages on target keywords:
1. Open each page and read it
2. Assess:
   - Is their content thorough or thin?
   - Does it have structured data / schema?
   - Is it up to date?
   - What format is it (list, guide, FAQ, comparison)?
   - What questions does it answer?
   - What questions does it NOT answer?
3. The gaps in competitor content become the content differentiation opportunities for SecurityBlogs
