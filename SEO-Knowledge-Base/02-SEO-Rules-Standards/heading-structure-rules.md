# Heading Structure Rules ‚Äî SecurityBlogs Standard

## Purpose

Heading tags (H1‚ÄìH6) serve two critical functions:
1. **SEO signal** ‚Äî Google uses heading structure to understand page topic hierarchy and keyword relevance
2. **User experience** ‚Äî Headings create scannable structure that helps readers navigate long-form content

Every page on SecurityBlogs and every client site must follow a strict heading hierarchy.

---

## Heading Hierarchy Rules

### H1 ‚Äî Page Title (One Per Page, Always)

**Rules:**
- Every page has exactly one H1 ‚Äî no more, no less
- The H1 contains the primary keyword for the page
- The H1 is the first heading a user sees
- The H1 should match or closely echo the meta title
- H1 is typically 5‚Äì10 words long
- Do not use the H1 as a button or decorative element

**Example (Service Page):** `Security SEO Services for Australian Security Companies`  
**Example (Blog Post):** `How to Rank #1 for "Security Camera Installation [City]"`  
**Example (Location Page):** `CCTV Installer SEO Services in Sydney`

---

### H2 ‚Äî Major Sections

**Rules:**
- H2s divide the page into major topic sections
- Each H2 represents a distinct sub-topic of the page's main subject
- H2s should contain secondary keywords or related keyword variations
- Aim for 3‚Äì6 H2s on a typical service page or blog post
- H2s should logically progress through the topic (awareness ‚Üí understanding ‚Üí application ‚Üí action)

**Example (on a Security SEO page):**
- `What Is Security SEO?`
- `Why CCTV and Access Control Companies Need Specialist SEO`
- `Our Security SEO Process`
- `Security SEO Results for Australian Clients`
- `Frequently Asked Questions`

---

### H3 ‚Äî Sub-Sections Within H2

**Rules:**
- H3s subdivide an H2 section into more specific points
- Use H3s when an H2 section has multiple distinct sub-topics
- Do not use H3s without a parent H2
- H3s can include long-tail keywords and specific queries

**Example (under the H2 "Our Security SEO Process"):**
- `Step 1: Technical SEO Audit`
- `Step 2: Security Keyword Research`
- `Step 3: On-Page Optimisation`
- `Step 4: Content Strategy`
- `Step 5: Link Building`

---

### H4‚ÄìH6 ‚Äî Deep Sub-Sections (Use Sparingly)

**Rules:**
- H4s subdivide H3 sections ‚Äî use only when genuinely needed
- H5 and H6 are rarely needed; avoid unless creating reference documentation
- Never skip levels (don't go from H2 to H4)

## Common Heading Mistakes

| ‚ùå Wrong | ‚úÖ Right |
|---|---|
| Multiple H1s on one page | Single H1 per page |
| H1 that doesn't include the primary keyword | H1 with primary keyword at or near the start |
| Using heading tags for visual styling only | Only use headings for genuine content hierarchy |
| Skipping from H1 to H3 | Always use H1 ‚Üí H2 ‚Üí H3 in order |
| All headings are questions | Mix of statement headings and question headings |
| Heading tags used for the navigation menu items | Navigation uses nav/ul elements, not heading tags |
| 15 H2s on a single page | 3‚Äì6 H2s with clear distinctions |

---

## Keyword Usage in Headings

### H1
- Include the exact primary keyword or a very close variant
- Do not force the keyword unnaturally ‚Äî if it doesn't sound right, rephrase

### H2
- Include secondary keywords and related terms
- Use natural phrasing ‚Äî headings should make sense to a human reader
- Include question-format headings for FAQ-style sections ("What is...?", "How does...?", "Why do...?")

### H3 and below
- Can include long-tail keywords
- Keep keyword inclusion natural ‚Äî never force keywords into subheadings

```

  H3: [Question 2]

```

### Blog Post Heading Template
```
H1: [Engaging Title with Primary Keyword]

H2: [Context-setting section ‚Äî why this matters]
H2: [Core concept explained]
  H3: [Sub-concept 1]
  H3: [Sub-concept 2]

  H3: [Step/Tip 2]
  H3: [Step/Tip 3]
```

## Heading Checklist

Before publishing any page, verify:
- [ ] Exactly one H1 on the page
- [ ] H1 includes the primary keyword
- [ ] H2s cover the major topic sections
- [ ] No heading levels are skipped
- [ ] Headings are used for structure, not visual decoration
- [ ] Question-format headings used for FAQ sections
- [ ] Heading hierarchy makes logical sense to a human reader
