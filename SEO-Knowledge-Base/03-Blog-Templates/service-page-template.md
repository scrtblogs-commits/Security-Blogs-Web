# Service Page Template — SecurityBlogs

## How to Use This Template

Copy this template for every new service page. Replace all `[BRACKETED]` items with real content. Follow all word count, keyword, and SEO rules from the 02-SEO-Rules-Standards folder.

---

## METADATA

```
Meta Title: [Primary Keyword] for [Target Client] | SecurityBlogs
(50–60 characters)

Meta Description: [Service name] specialist for [industry/client type]. [Key result/stat]. [CTA — "Book your free audit" or similar]. (140–158 characters)

URL Slug: /services/[service-slug]

Primary Keyword: [e.g., "Security SEO services"]
Secondary Keywords: [3–5 secondary keywords]
Schema: ProfessionalService + BreadcrumbList + FAQPage
```

### SECTION 1: INTRO / PROBLEM STATEMENT

**H2:** Why [Target Client Type] Need [Service Name]

Write 3–4 paragraphs that:
1. Describe the core problem this service solves (in the language of the buyer, not the agency)
2. Describe the consequence of NOT addressing this problem (lost leads, invisible to Google, AI ignoring you)
3. Introduce the SecurityBlogs solution at a high level
4. Transition into the service explanation
